# 惠电宿舍电费充值管理系统 - 系统设计说明书

## 1. 项目概述

### 1.1 项目背景
本系统是基于C/S架构的校园宿舍电费充值管理系统，旨在为高校提供便捷、高效的电费管理解决方案。

### 1.2 项目目标
- 实现学生电费余额查询与充值
- 实现管理员对学生信息、宿舍信息的管理
- 实现电费使用记录与充值记录的查询
- 提供统计分析功能

### 1.3 技术选型
- 开发框架：Qt 6.5.3
- 编程语言：C++17
- 数据库：SQLite
- 架构模式：C/S架构

## 2. 需求分析

### 2.1 功能需求

#### 2.1.1 用户登录
- 支持学生和管理员两种身份登录
- 密码加密存储
- 登录状态管理

#### 2.1.2 学生端功能
- 个人信息查看
- 电费余额查询
- 电费使用记录查询
- 在线充值功能
- 充值记录查询

#### 2.1.3 管理员端功能
- 学生信息管理（增删改查）
- 宿舍信息管理（增删改查）
- 为学生充值
- 电费扣费
- 系统统计（学生总数、总余额、总充值金额）
- 充值记录和用电记录查询

### 2.2 非功能需求
- 界面友好，操作简单
- 数据安全，密码加密
- 系统响应迅速
- 数据持久化存储

## 3. 系统架构设计

### 3.1 整体架构
采用C/S架构，客户端使用Qt框架开发，本地使用SQLite数据库存储数据。

### 3.2 模块划分
- 登录模块：负责用户身份认证
- 数据库管理模块：负责数据的增删改查
- 学生面板模块：学生端功能实现
- 管理员面板模块：管理员端功能实现

## 4. 数据库设计

### 4.1 数据表结构

#### 4.1.1 users表（用户表）
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INTEGER | 主键，自增 |
| username | TEXT | 用户名，唯一 |
| password | TEXT | 密码（SHA256加密） |
| name | TEXT | 姓名 |
| student_id | TEXT | 学号 |
| dormitory | TEXT | 宿舍号 |
| role | INTEGER | 角色（0-管理员，1-学生） |
| balance | REAL | 余额 |
| created_at | DATETIME | 创建时间 |

#### 4.1.2 dormitories表（宿舍表）
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INTEGER | 主键，自增 |
| dorm_number | TEXT | 宿舍号，唯一 |
| building | TEXT | 楼栋 |
| floor | INTEGER | 楼层 |
| current_balance | REAL | 当前余额 |
| last_reading | REAL | 上次读数 |
| last_update | DATETIME | 更新时间 |

#### 4.1.3 electricity_records表（电费使用记录表）
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INTEGER | 主键，自增 |
| user_id | INTEGER | 用户ID（外键） |
| dormitory | TEXT | 宿舍号 |
| usage | REAL | 用电量（kWh） |
| cost | REAL | 费用（元） |
| record_time | DATETIME | 记录时间 |
| remark | TEXT | 备注 |

#### 4.1.4 recharge_records表（充值记录表）
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INTEGER | 主键，自增 |
| user_id | INTEGER | 用户ID（外键） |
| student_id | TEXT | 学号 |
| dormitory | TEXT | 宿舍号 |
| amount | REAL | 充值金额 |
| balance_after | REAL | 充值后余额 |
| recharge_time | DATETIME | 充值时间 |
| operator_name | TEXT | 操作人 |

### 4.2 索引设计
- idx_users_username：用户名字段索引
- idx_users_dormitory：宿舍字段索引
- idx_dormitories_number：宿舍号索引
- idx_electricity_user：用户ID索引
- idx_electricity_dorm：宿舍号索引
- idx_recharge_user：用户ID索引
- idx_recharge_dorm：宿舍号索引

## 5. 核心类设计

### 5.1 DatabaseManager类
数据库管理类，采用单例模式，负责所有数据库操作。

主要方法：
- initDatabase()：初始化数据库
- addUser()：添加用户
- authenticateUser()：用户认证
- recharge()：充值
- deductElectricityCost()：电费扣费

### 5.2 StudentPanel类
学生面板类，实现学生端功能。

主要功能：
- 余额查询与显示
- 充值功能
- 用电记录查询
- 充值记录查询

### 5.3 AdminPanel类
管理员面板类，实现管理员端功能。

主要功能：
- 学生信息管理
- 宿舍信息管理
- 系统统计
- 记录查询

## 6. 界面设计

### 6.1 登录界面
- 用户名输入框
- 密码输入框
- 登录按钮
- 测试账号提示

### 6.2 学生端界面
- 顶部用户信息栏
- 标签页（余额查询、使用记录、充值记录）
- 刷新和退出登录按钮

### 6.3 管理员端界面
- 顶部管理员信息栏
- 标签页（系统统计、学生管理、宿舍管理、充值记录、用电记录）
- 各功能操作按钮

## 7. 安全设计

### 7.1 密码安全
- 使用SHA256算法加密存储密码
- 不在界面上明文显示密码

### 7.2 数据安全
- 使用事务保证数据一致性
- 充值和扣费操作使用事务处理
